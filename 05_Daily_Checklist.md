# ✅ Part 5: The Ultimate Daily Checklist / Главный ежедневный чеклист

### 🇬🇧 English
**Phase 1: Discovery & Research**
- [ ] Feed raw interview transcripts into AI to extract pain points and quotes.
- [ ] Run competitor analysis: Ask AI to summarize the UX flows of 3 direct competitors.
- [ ] Generate 3 divergent user personas using specific demographic constraints.

**Phase 2: Ideation & Wireframing**
- [ ] Use AI to generate an exhaustive list of edge cases for the main flow (e.g., "What if the user's internet drops here?").
- [ ] Generate component structures via v0.dev to visualize layout options.
- [ ] Apply AI auto-renaming and organization to keep Figma layers clean.

**Phase 3: High-Fidelity & UX Writing**
- [ ] Invoke your "UX Writer Skill" to review and rewrite all interface copy for clarity and tone.
- [ ] Generate dynamic empty states, error messages, and success toasts.
- [ ] Use generative AI to create non-generic, high-quality placeholder imagery.

**Phase 4: Handoff & QA**
- [ ] Generate a markdown documentation of the design decisions for the dev team.
- [ ] Run an AI accessibility audit on the final screens (color contrast, tap targets, aria-labels).
- [ ] Create testing scripts for QA based on the user flows.

### 🇷🇺 Русский
**Фаза 1: Исследования (Discovery)**
- [ ] Загрузить сырые транскрипты интервью в ИИ для извлечения болей (pain points) и цитат.
- [ ] Провести анализ конкурентов: попросить ИИ резюмировать UX-флоу 3 прямых конкурентов.
- [ ] Сгенерировать 3 различных персоны пользователя с заданными демографическими ограничениями.

**Фаза 2: Идея и Вайрфреймы**
- [ ] Использовать ИИ для генерации исчерпывающего списка краевых случаев (edge cases) (напр., "Что если тут пропадет интернет?").
- [ ] Сгенерировать структуры компонентов через v0.dev для визуализации опций лейаута.
- [ ] Применить ИИ для авто-переименования и организации слоев в Figma.

**Фаза 3: High-Fidelity и UX-копирайтинг**
- [ ] Вызвать "Скилл UX-писателя" для проверки и переписывания всего текста в интерфейсе для ясности и тона.
- [ ] Сгенерировать динамичные пустые состояния (empty states), сообщения об ошибках и успехах.
- [ ] Использовать генеративный ИИ для создания уникальных, качественных изображений-плейсхолдеров.

**Фаза 4: Передача в разработку и QA (Handoff)**
- [ ] Сгенерировать markdown-документацию с описанием дизайн-решений для команды разработки.
- [ ] Запустить ИИ-аудит доступности финальных экранов (контраст цветов, зоны клика, aria-лейблы).
- [ ] Создать сценарии тестирования для QA на основе пользовательских флоу.
